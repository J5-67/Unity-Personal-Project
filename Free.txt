기능 정의서
1.     게임 개요
이름 	FREE()
장르 	3D 횡스크롤 하드코어 액션 플랫포머
타겟	 PC (Windows) / Console (PS5, Xbox)
태그	#3D #횡스크롤 #액션 #선형 #정밀 플랫폼 #고난이도 
2.     시스템 구조 & 기술적 구현
[메인 메뉴 & 설정]
●	게임 시작
○	새로운 게임 : 데이터 초기화 후 1번 맵 로드
○	이어하기 : 로컬 저장된 데이터(JSON)를 로드하여 체크포인트에서 시작
●	설정
○	설정은 PlayerPrefs로 저장
○	그래픽 설정 : 전체화면 / 창모드 (해상도 조절)
○	소리 설정 : Audio Mixer 연동
■	Master / BGM / SFX : Slider UI 로 0~100% 조절 / Input Field 입력
○	마우스 감도 설정 : Slider UI 로 0~100% 조절 / Input Field 입력
●	게임 종료 
○	Application.Quit() 처리
[일시 정지]
●	진입 방법 : 인게임 플레이 중 ESC 키 입력
●	기능
○	게임 내 시간 정지 (Time.timeScale = 0)
○	플레이어 조작 입력 차단 (UI 조작만 허용)
■	PlayerInput 컴포넌트의 SwitchCurrentActionMap("UI") 기능을 사용하여 인게임 액션은 막고 UI 조작만 활성화
○	배경 흐림 처리
●	메뉴 구성
○	계속하기 : 메뉴를 닫고 게임 시간 재개
○	설정 : 설정 팝업 출력 (메인 메뉴의 설정과 동일)
○	저장 : 현재 상태(위치, HP, 스택 등)를 JSON 파일로 저장
○	메인 메뉴 : 메인 메뉴로 돌아가기
[맵 로딩]
●	비동기 로드
○	게임 시작 / 맵 이동 시 검은 화면 대신 로딩 컷신 또는 진행률 바 출력
○	최적화: SceneManager.LoadSceneAsync를 통한 비동기 로딩 구현
○	메모리 관리: 씬 전환 시 Resources.UnloadUnusedAssets() 및 GC.Collect()를 호출하여 이전 맵의 미사용 리소스 즉시 해제
[데이터 관리]
●	저장 / 불러오기: 플레이어 위치, HP, 대화 진행도, 처리한 적 등을 JSON 파일로 저장
[최적화]
●	비동기 에셋 관리 : Addressable Asset System을 사용하여 맵 데이터와 프리팹 로드
●	오브젝트 풀링 : Pooling Manager을 구현하여 재사용성 극대화
________________________________________
[컷씬]
●	타임라인
○	카메라 이동, 캐릭터 애니메이션, VFX, 사운드를 타임라인 트랙으로 제어
○	인게임 플레이 ↔ 컷씬 전환 시 자연스러운 블렌딩 처리 (Cinemachine)
[대화 시스템]
●	UI 구성
○	하단 대화창 패널 + 화자 이름 + 화자 일러스트 + 대사 텍스트
●	데이터 구조
○	CSV 파일 파싱 (ID, 화자, 표정, 대사 내용, 다음 연결 ID)
●	출력 연출
○	타자기 효과 : 한 글자씩 Delay를 주며 출력 (코루틴 사용)
○	출력 소리 : 글자 출력 시 효과음 재생
●	편의 기능
○	빠른 출력: 클릭 / 키 입력 시 현재 대사 즉시 전체 출력
○	스킵 : 특정 키 홀드 시 대화 이벤트 종료
________________________________________
[상태 관리 시스템]
●	목적 : 플레이어와 적의 복잡한 분기 로직 제어 및 코드 유지보수성 확보
●	구조 : IState 인터페이스(Enter, Execute, Exit)를 기반으로 상태를 클래스로 캡슐화
[Hook (와이어 액션)]
●	메커니즘 : 마우스 커서 위치로 로프 발사 (Raycast). 레이어 / 태그를 판별하여 효과 분기
○	당겨오기 : 소형 적, 소형 오브젝트 -> 플레이어 쪽으로 날아옴 (기절 상태)
○	이동하기 : 벽, 천장, 대형 적, 정지 상태의 적 -> 플레이어가 대상 쪽으로 이동
●	자원 : 쿨타임 없음 (무제한 사용)
●	시너지 : 적중 성공 시 Dash 스택 +1 즉시 충전
●	Hook 이동 메커니즘 : 타겟 방향으로 가속도를 적용하되, 해제 시점의 관성을 보존하여 유동적인 공중 제어가 가능하도록 구현.
●	Hook 비주얼 시뮬레이션 : 무거운 물리 컴포넌트 대신, 가벼운 수식 기반의 '벨렛 적분' 알고리즘을 적용. 로프의 장력, 중력에 의한 처짐, 이동 관성에 따른 흔들림을 시각적으로만 리얼하게 시뮬레이션하여 퍼포먼스와 비주얼 두 마리 토끼를 잡음.
[Dash (돌진 & 관통)]
●	메커니즘: 마우스 커서 방향으로 짧고 강력하게 이동 / 경로상의 적을 통과
●	자원: 스택제 (최대 2개), 시간 경과 시 자동 충전
●	상태 이상 [정지]
○	Dash에 관통당한 적은 즉시 정지 상태가 됨 (위치 / 회전 고정, 중력 영향 없음)
○	정지된 적은 Hook 사용 시 벽으로 판정 변경
○	3초 후, 정지 풀리며 폭발 데미지(즉사급) 발생
[이동]
●	기본
○	좌우 이동, 점프, 드랍, 빠른 낙하
●	조작감 보정
○	가변 점프 : 버튼 누르는 시간에 비례해 점프 높이 조절
○	코요테 타임 : 절벽에서 떨어져도 아주 잠깐 점프 허용
○	입력 버퍼 : Queue 또는 링 버퍼 자료구조를 활용. 착지 0.x초 전 입력된 점프 명령을 저장했다가 착지 즉시 실행
●	특수
○	벽 타기
○	벽 차기
[컨트롤러]
●	콘솔 게이머를 위한 컨트롤러 입력 지원
○	조준 보정 기능 제공
[업적]
●	플레이어의 행동을 추적하여 특정 행동을 할 시 리워드 제공
________________________________________
[전투 시스템]
●	HP
○	낮은 최대 체력 (1 ~ 3)
○	피격 시 1~2초간 무적 (피격 연출)
●	사망 처리
○	HP 0 또는 낙사 구간 접촉 시 사망
○	가장 마지막 체크포인트에서 리스폰
●	피드백
○	히트 스탑 : 타격 / 피격 순간 0.05~0.1초간 시간 정지
○	화면 흔들림 : Dash 관통, 폭발, 피격 시 카메라 진동 (Cinemachine Impulse)
[적 AI]
●	적(Enemy) AI 로직
○	[대기] ↔ [순찰] ↔ [추격] 기본 순환
○	[정지] : 피격 시 진입, 물리 연산 중단 및 쉐이더 교체
○	[폭발] : Exit() 시점에 파티클 재생 후 오브젝트 풀 반환
________________________________________
[UI / UX]
●	마우스 커서
○	플레이어 -> 마우스 커서 방향으로 가는 점선 표시 
■	점선의 길이는 벽에 닿거나 최대 사거리까지 표시
○	레이어 / 태그를 판별하여 색 변경하거나 커서 변경
■	평상시 : 민트색 / 조준선
■	사거리 내 벽 / 대형 적 조준 시 : 녹색 (이동 가능 표시) / 가는 화살표
■	사거리 내 소형 적 조준 시 : 빨간색 (당기기 가능 표시) / 오는 화살표
●	HUD
○	피격시에만 잠깐 플레이어 주변에 HP 현황 출력
○	마우스 커서 주변 또는 하단: Dash 스택 (슬롯 형태 UI)
[쉐이더그래프]
●	글리치 이펙트 : Time 변수에 따른 노이즈 함수를 사용하여 시각적 효과 연출
